## 1.3 Cloud Concepts and Connectivity

### Summary
Cloud computing provides on-demand access to computing resources such as servers, storage, networking, and applications. Cloud environments are designed to scale up or down as needed, support access from anywhere, and allow multiple customers to share the same underlying infrastructure through multitenancy.

### Key Terms
- **Cloud computing:** On-demand access to computing resources over a network.
- **Elasticity:** The ability to scale resources up or down as needed.
- **Scalability:** The ability to support growth, such as more users, applications, or workloads.
- **Multitenancy:** When multiple customers use the same cloud infrastructure while keeping their environments separate.
- **Virtual network:** A software-based network created inside a virtualized or cloud environment.
- **NFV:** Network Function Virtualization; replacing physical network devices with virtual versions.
- **Hypervisor:** Software that allows multiple virtual machines to run on one physical system.
- **VPC:** Virtual Private Cloud; a pool of private cloud resources created inside a public cloud.
- **Transit gateway:** A cloud routing service used to connect VPCs and networks together.
- **VPN:** Virtual Private Network; an encrypted connection across a public network.
- **Internet gateway:** A cloud gateway that allows resources to communicate with the internet.
- **NAT gateway:** A gateway that allows private cloud resources to access external resources while preventing direct inbound access.
- **VPC endpoint:** A direct private connection to services within a cloud provider network.
- **Security group:** A cloud firewall rule set usually applied to a specific virtual network interface or resource.
- **Network security list:** A set of cloud security rules usually applied broadly to an entire subnet.
- **CIDR:** Classless Inter-Domain Routing; notation used to define IP address ranges.

### Important Details

#### Designing the Cloud
- Cloud computing provides on-demand computing power.
- Resources can often be deployed with the click of a button.
- Cloud environments support elasticity, meaning resources can scale up or down as needed.
- Applications can also scale to support large implementations.
- Cloud services are commonly accessible from anywhere with the right network connection and permissions.
- Multitenancy allows many different clients to use the same cloud infrastructure while keeping their data and resources separated.

#### Virtual Networks
- A traditional server farm may contain many physical servers, switches, and routers.
- These servers are usually connected with enterprise switches and routers, often with redundancy for availability.
- Virtualization allows many physical servers to be migrated into virtual servers running on fewer physical systems.
- For example, 100 physical servers could potentially be replaced by one powerful physical server running 100 virtual servers.
- Virtual networks allow these virtual servers to communicate as if they were connected to physical network equipment.

#### Network Function Virtualization
- **NFV**, or Network Function Virtualization, replaces physical network devices with virtual versions.
- These virtual devices can be managed through a hypervisor or cloud management platform.
- Virtual network devices can provide the same functions as physical devices, such as:
  - Routing
  - Switching
  - Load balancing
  - Firewalling
- NFV allows network functions to be deployed quickly and easily.
- Deployment options may include virtual machines, containers, and fault-tolerant designs.

#### Connecting to the Cloud
- A **VPC**, or Virtual Private Cloud, is a pool of resources created inside a public cloud.
- It is common to create multiple VPCs for different applications, environments, or business needs.
- VPCs can be connected using a transit gateway, which acts like a cloud router.
- Users and networks can connect to VPCs through secure connections such as VPNs.
- VPCs commonly use different IP subnets to separate resources.
- A site-to-site VPN can connect an on-premises network to a cloud VPC over the internet.
- An internet gateway allows cloud resources to communicate with users or services on the internet.
- A NAT gateway allows private cloud subnets to access external resources while preventing external resources from directly accessing the private subnet.
- A VPC endpoint provides a direct private connection between resources and services within the cloud provider’s network.

#### Security Groups and Network Security Lists
- Cloud security groups and security lists act like firewalls for cloud environments.
- They control inbound and outbound traffic flows.
- Rules can be based on Layer 4 information, such as TCP or UDP port numbers.
- Rules can also be based on Layer 3 information, such as individual IP addresses or CIDR blocks.
- Security rules may support both IPv4 and IPv6.

#### Network Security Lists
- Network security lists apply security rules to an entire IP subnet.
- These rules affect all devices in that subnet.
- Security lists are broad and can become difficult to manage.
- Not all devices in the same subnet have the same security needs.
- More granular security control may be needed in many cloud environments.

#### Network Security Groups
- Network security groups apply security rules to a specific virtual network interface or resource.
- They are more granular than network security lists.
- Different devices in the same IP subnet can have different security rules.
- Network security groups provide better control and are often preferred for cloud security rule management.

### Example
A company may create multiple VPCs in a public cloud for different applications. Each VPC may use its own subnet. A transit gateway can connect the VPCs together, while a site-to-site VPN can connect the company’s office network to the cloud. A NAT gateway can allow private cloud servers to reach the internet for updates without allowing direct inbound access. Security groups can then control which systems are allowed to communicate with each cloud resource.

## 1.3 Cloud Models

### Summary
Cloud models describe how cloud resources are deployed and how services are provided to users. Deployment models include public, private, and hybrid cloud. Service models include SaaS, IaaS, and PaaS.

### Key Terms
- **Public cloud:** Cloud services available to customers over the internet.
- **Private cloud:** A cloud environment dedicated to one organization, often similar to a virtualized local data center.
- **Hybrid cloud:** A mix of public and private cloud environments.
- **SaaS:** Software as a Service; complete software applications delivered on demand over the internet.
- **IaaS:** Infrastructure as a Service; cloud-based infrastructure such as servers, storage, and networking.
- **HaaS:** Hardware as a Service; another way to describe outsourcing physical infrastructure.
- **PaaS:** Platform as a Service; a cloud platform used to develop and deploy applications without managing the underlying infrastructure.

### Important Details

#### Cloud Deployment Models
- **Public cloud** services are available to customers over the internet.
- Public cloud resources are owned and operated by a cloud provider.
- **Private cloud** is dedicated to one organization.
- A private cloud can be thought of as an organization’s own virtualized local data center.
- **Hybrid cloud** combines public and private cloud resources.
- Hybrid cloud allows organizations to use both local/private resources and public cloud services.

#### Software as a Service
- **SaaS**, or Software as a Service, provides on-demand software.
- SaaS applications usually do not require local installation.
- The cloud provider manages the application, infrastructure, updates, and availability.
- SaaS can reduce the need for organizations to manage their own services, such as email or payroll systems.
- SaaS provides central management of data and applications.
- With SaaS, the user’s data is stored with the provider, so trust and security are important.
- SaaS is a complete application offering and usually requires no development work from the customer.
- Examples include Google Mail and Microsoft 365.

#### Infrastructure as a Service
- **IaaS**, or Infrastructure as a Service, provides cloud-based infrastructure.
- IaaS may also be called **Hardware as a Service**, or HaaS.
- IaaS allows organizations to outsource equipment such as servers, storage, and networking hardware.
- The cloud provider manages the physical hardware and data center.
- The customer is still responsible for managing the operating systems, applications, configurations, and security.
- With IaaS, the data is still stored in the cloud, but the customer usually has more control than with SaaS.
- Web server hosting providers are a common example of IaaS.

#### Platform as a Service
- **PaaS**, or Platform as a Service, provides a cloud platform for developing and deploying applications.
- With PaaS, the customer does not manage servers, software maintenance, hardware, HVAC, or the underlying platform.
- The provider handles the platform, while the customer focuses on application development.
- The customer does not have direct control over the underlying data center, physical infrastructure, or support staff.
- Security and platform maintenance are handled by the provider, so choosing a trusted provider is important.
- PaaS allows developers to build applications using tools and services available on the platform.
- Salesforce is an example of a platform-based cloud service.

### Example
A company may use SaaS for email through Microsoft 365, IaaS to host a web server in the cloud, and PaaS to build a custom business application without managing the underlying servers. If the company keeps some systems in its own data center while also using public cloud resources, it is using a hybrid cloud model.
