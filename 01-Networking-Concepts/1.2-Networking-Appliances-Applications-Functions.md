## 1.2 Networking Devices

### Summary
Networks use different appliances and technologies to move traffic, protect systems, provide wireless access, balance workloads, and store shared data. Each device has a specific role, and many modern devices can perform multiple functions.

### Key Terms

- **Router:** A device that forwards traffic between different IP networks or subnets.
- **Switch:** A device that connects devices within the same local network, usually operating at Layer 2.
- **Layer 3 switch:** A switch that also has routing capabilities.
- **Firewall:** A security device or application that filters traffic based on rules, ports, protocols, or applications.
- **NGFW:** Next-Generation Firewall; a firewall with advanced features such as application control, deep inspection, and threat prevention.
- **IDS:** Intrusion Detection System; detects suspicious or malicious activity and generates alerts.
- **IPS:** Intrusion Prevention System; detects and blocks suspicious or malicious activity before it reaches the network.
- **IDPS:** Intrusion Detection and Prevention System; combines IDS and IPS functions.
- **Load balancer:** A device or application that distributes traffic across multiple servers.
- **Proxy:** A system that sits between a user and an external network, forwarding requests on the user’s behalf.
- **NAS:** Network Attached Storage; shared storage connected to the network.
- **SAN:** Storage Area Network; dedicated storage network that appears more like local storage to servers.
- **Access Point:** A wireless device that allows clients to connect to a wired network.
- **Wireless LAN Controller:** A system used to centrally manage multiple wireless access points.

### Important Details

#### Routers
- Routers send data from one IP subnet to another.
- Routers commonly connect different network types, such as LANs and WANs.
- Routers operate mainly at **Layer 3**, using IP addresses to make forwarding decisions.
- Some switches include routing functionality, making them **Layer 3 switches**.

#### Switches
- Switches are a core part of enterprise networks.
- Traditional switches operate at **Layer 2**, using MAC addresses to forward frames.
- Switches often use **ASICs**, or Application-Specific Integrated Circuits, for fast hardware-based forwarding.
- Some switches provide **PoE**, or Power over Ethernet, which can power devices such as phones, cameras, and access points.
- A switch with routing capabilities is called a **Layer 3 switch**.

#### Firewalls
- Firewalls filter traffic based on rules, such as port number, protocol, source/destination address, or application.
- Traditional firewalls commonly filter traffic using Layer 3 and Layer 4 information.
- Next-Generation Firewalls can inspect and control applications, not just ports and IP addresses.
- Firewalls often sit at the **ingress/egress** points of a network, meaning where traffic enters and leaves.
- Firewalls may also perform routing, NAT, VPN encryption, and dynamic routing.
- **NAT**, or Network Address Translation, allows private internal addresses to communicate with external networks using public addresses.

#### IDS and IPS
- IDS stands for **Intrusion Detection System**.
- IPS stands for **Intrusion Prevention System**.
- An intrusion is an attempted exploit against a system, application, or network.
- Examples of attacks include buffer overflows, cross-site scripting, malware activity, and suspicious traffic patterns.
- **Detection** means the system identifies suspicious activity and sends an alert.
- **Prevention** means the system blocks the suspicious activity before it reaches the target.
- Some systems combine both functions and are called **IDPS**.

#### Load Balancers
- Load balancers distribute network traffic across multiple servers.
- They are commonly used in web server farms, database farms, and large-scale applications.
- Load balancers improve availability by detecting server outages and redirecting traffic to healthy servers.
- They can improve fault tolerance and performance.
- Some load balancers can cache data.
- Load balancers may also perform **TCP offloading** and **SSL/TLS offloading**, reducing the workload on backend servers.
- They can help prioritize different types of traffic.

#### Proxies
- Proxies sit between users and external networks.
- A proxy receives a user’s request, forwards it on their behalf, and then returns the response.
- Proxies can be used for caching, access control, URL filtering, and content scanning.
- Proxies can also help hide internal client information from external systems.

#### NAS and SAN
- **NAS**, or Network Attached Storage, provides shared file storage over the network.
- **SAN**, or Storage Area Network, provides block-level storage that appears more like local storage to servers.
- SANs are often better for workloads that require frequent small changes to stored data.
- NAS and SAN solutions both require high bandwidth.
- Storage traffic is often placed on its own isolated network for performance and security.

#### Access Points
- An Access Point provides wireless connectivity to network clients.
- It allows devices such as laptops, phones, and tablets to communicate with the wired network.
- Access points usually operate at **Layer 2**.
- An AP bridges wireless clients to the wired network.
- An access point is not always the same as a wireless router. A home wireless router usually combines multiple functions, including routing, switching, firewalling, and wireless access.

#### Wireless LAN Controllers
- Wireless LAN Controllers centrally manage multiple access points.
- They are useful in organizations with many APs across one or more locations.
- They can deploy new access points.
- They can monitor wireless performance and security.
- They can push configuration changes to many APs at once.
- They can create reports about access point usage and wireless clients.

### Example
In a company network, a user’s laptop may connect wirelessly to an access point. The access point bridges the traffic to the wired network. A switch forwards the traffic inside the LAN. A router or Layer 3 switch moves traffic between subnets. A firewall filters traffic before it leaves the company network. If the user is accessing a company web application, a load balancer may distribute the request to one of several backend servers.
